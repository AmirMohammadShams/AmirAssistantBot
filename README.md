# مستندات جامع ربات معامله‌گر واگرایی مثبت (Bullish Divergence Crypto Bot)

این سند شامل توضیحات کامل درباره نحوه کارکرد، معماری، استراتژی و نحوه تنظیمات ربات معامله‌گر ارز دیجیتال است.

---

## ۱. معرفی و نمای کلی (Overview)
این ربات یک سیستم الگوریتمی تمام‌خودکار (Automated Algorithmic Trading Bot) است که وظیفه دارد بازار ارزهای دیجیتال (در صرافی بایننس یا بای‌بیت) را به صورت ۲۴ ساعته رصد کند. هدف اصلی این ربات، پیدا کردن الگوهای **واگرایی مثبت (Bullish Divergence)** در تایم‌فریم یک ساعته (1H) و ورود به معاملات خرید (Long) بر اساس مدیریت ریسکِ از پیش‌تعیین‌شده است.

---

## ۲. معماری و ساختار فنی (Architecture & Tech Stack)
- **زبان برنامه‌نویسی:** Python 3.12+
- **اتصال به صرافی:** کتابخانه قدرتمند `ccxt` (پشتیبانی از Binance, Bybit و ده‌ها صرافی دیگر).
- **تحلیل داده‌ها:** کتابخانه‌های `pandas` و `numpy` برای پردازش داده‌های بازار.
- **اندیکاتورهای تکنیکال:** کتابخانه `pandas-ta` جهت محاسبه دقیق RSI و میانگین متحرک (EMA).
- **زمان‌بندی (Scheduling):** کتابخانه `schedule` برای بیدار شدن دقیق ربات در ثانیه ۵ ام هر ساعت.
- **اطلاع‌رسانی (Logging):** ثبت لحظه‌ای اتفاقات در فایل `bot.log` و ارسال همزمان لاگ‌ها به **تلگرام** کاربر از طریق توکن اختصاصی.

## ۳. استراتژی معاملاتی (Trading Strategy)

ربات بر اساس ۴ شرط اصلی معامله می‌کند:

### الف) تایم‌فریم و داده‌ها
ربات هر ساعت یک‌بار بیدار شده و **۲۵۰ کندل ۱ ساعته‌ی اخیر** را برای ده‌ها ارز مختلف (مانند BTC, ETH, SOL, GALA, AAVE و ...) دانلود می‌کند.

### ب) اندیکاتورها
- **RSI (شاخص قدرت نسبی):** با دوره ۱۴ (Length = 14)
- **EMA (میانگین متحرک نمایی):** با دوره ۲۰۰ (Length = 200)

### ج) تشخیص واگرایی مثبت (Bullish Divergence)
ربات به دنبال "دره‌ها" (Swing Lows) یا کف‌های محلی می‌گردد (کف‌هایی که ۵ کندل قبل و ۵ کندل بعد از آن‌ها بالاتر باشند). اگر دو کفِ آخر شرایط زیر را داشته باشند، واگرایی تایید می‌شود:
1. **شرط قیمتی:** قیمت در کفِ دوم باید پایین‌تر از کفِ اول باشد ($P_2 < P_1$).
2. **شرط اسیلاتور:** عدد اندیکاتور RSI در کفِ دوم باید بالاتر از کفِ اول باشد ($RSI_2 > RSI_1$).
*(این یعنی قیمت در حال ریزش است اما قدرت فروشندگان در حال کاهش است، که نویدبخش یک صعود است).*

### د) فیلتر روند (Trend Filter - EMA 200)
برای جلوگیری از خرید در زمان‌هایی که بازار در حال سقوط آزاد (سقوط سهمگین) است، ربات قیمت فعلی را با خط EMA 200 روزه مقایسه می‌کند. 
اگر قیمت **فاصله‌ی رو به پایین بسیار زیادی** (بیش از ۵ درصد) با خط EMA 200 داشته باشد، ربات سیگنال را اصطلاحاً Ignore (نادیده) می‌کند تا چاقوی در حال سقوط را نگیرد. (این ویژگی در کدهای ربات قابل خاموش کردن است).

## ۴. مدیریت ریسک و سفارشات (Risk & Order Management)

ربات دارای یک سیستم مدیریت سرمایه بسیار پیشرفته است:

1. **ریسک ثابت (Fixed Risk):** ربات طوری برنامه‌ریزی شده که در هر معامله دقیقاً **۱ درصد** (یا هر عددی که در تنظیمات مشخص کنید) از کل سرمایه (Equity) شما را به خطر بیندازد.
2. **محاسبه هوشمند حجم (Position Sizing):** ربات فاصله نقطه ورود تا حد ضرر را محاسبه کرده و دقیقاً بر همان اساس حجم معامله (Position Size) را طوری تعیین می‌کند که اگر حد ضرر خورد، شما فقط همان ۱ درصد را از دست بدهید.
3. **حد ضرر (Stop Loss):** حد ضرر دقیقاً زیرِ کفِ دوم (پایین‌ترین نقطه واگرایی) بعلاوه ۰.۲٪ بافر قرار می‌گیرد تا شدوهای ناگهانی باعث بسته شدن معامله نشوند.
4. **حد سود (Take Profit):** ربات از نسبت ریسک به ریوارد **۱ به ۱.۵** (R:R = 1:1.5) استفاده می‌کند.

## ۵. راهنمای فایل تنظیمات (`.env`)

برای شخصی‌سازی ربات، شما یک فایل `.env` (مخفی) در کنار سورس ربات می‌سازید و مقادیر زیر را در آن قرار می‌دهید:

```ini
# صرافی مورد نظر (binance یا bybit)
EXCHANGE_ID=binance

# کلیدهای API صرافی (اگر خالی بگذارید، ربات حالت Simulation خواهد داشت)
API_KEY=your_api_key_here
API_SECRET=your_api_secret_here

# روشن بودن حالت آزمایشی صرافی (True یعنی از پول مجازی صرافی استفاده شود)
TESTNET=true

# لیست ارزها (جدا شده با کاما)
TRADING_PAIRS=BTC/USDT,ETH/USDT,SOL/USDT,...

# درصد ریسک در هر معامله (مثلا 1.0 یعنی 1 درصد کل سرمایه)
RISK_PERCENTAGE=1.0

# فیلتر خط روند 200 روزه (True یعنی روشن، False یعنی خاموش)
USE_EMA_FILTER=true

# تنظیمات تلگرام (برای ارسال خودکار گزارش‌ها)
TELEGRAM_TOKEN=توکن_ربات_تلگرام_شما
TELEGRAM_CHAT_ID=آیدی_عددی_شما

# تنظیمات پراکسی (برای دور زدن تحریم‌های بایننس و فیلترینگ تلگرام روی سرور ایران)
HTTP_PROXY=socks5h://127.0.0.1:10808
```

## ۶. نحوه اجرا و استقرار در سرور (Deployment)

برای اجرای این ربات در سرور لینوکس (VPS)، دستورات زیر در ترمینال اجرا می‌شوند:

1. **ساخت محیط ایزوله (Virtual Environment):**
   ```bash
   python3 -m venv venv
   source venv/bin/activate
   ```
2. **نصب نیازمندی‌ها (کتابخانه‌ها):**
   ```bash
   pip install -r requirements.txt
   pip install pysocks  # ضروری برای اتصال پراکسی
   ```
3. **اجرای ربات در پس‌زمینه (Screen یا Nohup):**
   برای اینکه ربات با بسته شدن ترمینال متوقف نشود، معمولاً آن را با ابزارهایی مثل `screen` اجرا می‌کنند:
   ```bash
   screen -S crypto_bot
   python bot.py
   ```
   *(سپس با فشردن کلیدهای `Ctrl + A` و سپس `D` ربات در پس‌زمینه سرور شما به کار خود تا ابد ادامه خواهد داد).*
